# DynamicIslandBar 上/左/右吸附与底部中心卡片同构设计

> Status: Approved design spec for the next implementation planning step.
> Date: 2026-07-04
> Project: `DynamicIslandBar`

## 1. 目标

把 `DynamicIslandBar` 的 `顶部吸附`、`左侧吸附`、`右侧吸附` 收敛到和 `底部胶囊` 同一套功能模型上，避免继续维护“底部一套逻辑、侧边一套补丁逻辑”的分叉实现。

这次设计重点覆盖：

- 上 / 左 / 右吸附与底部胶囊的功能对齐
- 中心卡片在四种吸附模式下的统一展示模型
- 左右吸附模式的竖向排版规则
- 中心卡片歌词常态、悬浮详细态、播放器控件、详细页展开方向
- 上 / 左 / 右吸附的默认参数与长度调节规则

这次设计不覆盖：

- 新的视觉主题
- 新的音乐控制能力
- 多显示器独立吸附规则
- 重新设计整个胶囊外观系统

## 2. 产品意图

用户当前对吸附模式的预期非常明确：

1. `底部胶囊` 是完整功能基准。
2. `顶部吸附` 不是一个独立产品形态，只是“默认参数沿用顶部模式”的底部同构版本。
3. `左 / 右吸附` 也不是简化版，它们必须具备底部胶囊已有的功能、参数与交互，只是阅读方向改为竖向。

所以，这次实现不应继续把侧边模式当作“底部功能裁剪版”处理，而应把四种吸附模式理解为：

- `底部`：标准横向布局
- `顶部`：标准横向布局，默认参数不同
- `左侧`：标准功能模型 + 竖向布局
- `右侧`：标准功能模型 + 竖向布局

## 3. 用户可见行为

### 3.1 顶部吸附

顶部吸附与底部胶囊保持同一套功能和交互。

行为规则：

- 中心卡片常态显示歌词跑马灯
- 悬浮中心卡片显示歌曲名、歌手、播放器控件等详细态
- 长度和中心卡片参数都支持和底部一样的调节
- 只在默认初始值上沿用顶部模式参数

顶部吸附不做竖排改造。

### 3.2 左右吸附

左右吸附必须具备与底部胶囊一致的功能能力，但内容改为竖向适配。

行为规则：

- 主胶囊仍保留应用区、中心卡片区、系统区这三段结构
- 应用图标保持正向，不旋转图标本身
- 元素改成竖向排列
- 中心卡片常态显示歌词
- 歌词从下往上滑动
- 悬浮详细态显示歌曲名、歌手、播放器控件
- 详细页始终朝胶囊外侧展开：
  - 左侧吸附向右展开
  - 右侧吸附向左展开

### 3.3 默认参数与长度规则

`顶部 / 左侧 / 右侧` 三种吸附模式在首次吸附时，默认长度沿用顶部模式的参数配置。

但默认值只决定初始状态，不代表锁定外观。

吸附后仍需支持：

- 用户继续手动调节胶囊长度
- 用户继续手动调节中心卡片长度比例
- 用户继续沿用底部已有的外观调节能力

换句话说，`顶部参数` 只是上 / 左 / 右三种吸附模式的初始模板，不是这些模式的永久限制。

## 4. 统一中心卡片模型

### 4.1 统一状态机

中心卡片在四种吸附模式下都应来自同一套状态判断，而不是为侧边模式再维护单独的文案和展示策略。

统一的核心状态：

1. `Hidden`
   没有锁定应用时隐藏中心卡片内容。
2. `Lyrics`
   音乐应用在常态下显示歌词。
3. `Details`
   鼠标悬浮时显示详细态，包括歌曲名、歌手、播放器控件。
4. `AppDetails`
   非音乐应用显示应用标题、状态、副操作。

判断原则：

- 是否为音乐应用，决定是否进入音乐展示模型
- 是否悬浮，决定音乐应用显示 `Lyrics` 还是 `Details`
- 吸附方向只影响布局方向，不改变状态语义

### 4.2 常态歌词规则

音乐应用在常态下，中心卡片必须优先展示歌词，不应把应用图标或控制按钮放在主阅读区中央遮挡歌词空间。

规则：

- 底部 / 顶部：歌词沿横向从右往左滑动
- 左侧 / 右侧：歌词沿竖向从下往上滑动
- 应用图标固定在歌词卡片的起始端：
  - 横向模式在左侧
  - 竖向模式在顶部
- 歌词是常态下的主内容

### 4.3 悬浮详细态规则

详细态必须和底部胶囊功能一致，不允许侧边模式删减元素。

详细态最低要求：

- 应用图标
- 歌曲名
- 歌手
- 播放 / 暂停
- 上一首 / 下一首
- 音量
- 其他已存在于底部的播放相关控制项

布局规则：

- 底部 / 顶部：横向详细页
- 左侧 / 右侧：竖向详细页

详细态的视觉表现可以随方向重排，但字段、按钮、能力和参数不能缺失。

## 5. 布局结构

### 5.1 不再引入第二套侧边中心卡片系统

当前代码里存在“侧边专用重排”和“歌词层 / 详细层的额外分支解释”，这是问题根源。

新的实现边界应是：

- 只保留一套中心卡片展示模型
- 只保留一套中心卡片参数来源
- 把差异收束到“布局方向”和“动画方向”上

允许保留歌词层与详细层两个视觉容器，但它们必须共享同一套语义字段和状态来源，而不是各自独立决定要显示什么。

### 5.2 四种吸附模式的结构映射

统一结构：

- 应用区
- 分隔区
- 中心卡片区
- 分隔区
- 系统区

方向映射：

- 底部：从左到右排列
- 顶部：从左到右排列
- 左侧：从上到下排列
- 右侧：从上到下排列

这意味着：

- 左右吸附不是创建新的“迷你纵向版”胶囊
- 只是把底部同一套槽位顺序映射到纵向主轴

### 5.3 详细页展开方向

所有带悬浮详细页的元素都必须沿胶囊外侧展开。

规则：

- 底部吸附：向上展开
- 顶部吸附：向下展开
- 左侧吸附：向右展开
- 右侧吸附：向左展开

这条规则适用于：

- 中心卡片详细态相关浮层
- 应用悬浮详情
- 其他依附在胶囊上的悬浮详细页

## 6. 参数与交互

### 6.1 长度调节

`CenterCardWidthPercent` 的语义仍然保持为“中心卡片占当前胶囊主轴长度的比例档位”。

映射规则：

- 底部 / 顶部：映射到中心卡片宽度
- 左侧 / 右侧：映射到中心卡片高度

用户拖拽长度时：

- 横向模式响应水平拖拽
- 竖向模式响应垂直拖拽

### 6.2 边界约束

胶囊必须保留屏幕内可操作约束，避免拖拽后主体跑出屏幕导致不可见或不可恢复。

最低要求：

- 主体不能完全拖出屏幕
- 吸附前后的自由态位置都需被约束
- 左右吸附释放后恢复自由态时，应从当前可见位置继续拖动

### 6.3 默认参数继承

上 / 左 / 右三种吸附模式进入吸附状态时，默认继承顶部模式参数：

- 长度
- 粗细
- 玻璃质感
- 发光
- 中心卡片初始尺寸规则

但保存和更新仍使用各自当前模式下的可调结果，而不是每次强制回退到顶部参数。

## 7. 实现边界

这次实现应优先修改以下边界，而不是在 `MainWindow` 中继续增加临时条件分支：

- `CenterCardPresentationPolicy`
  统一中心卡片状态语义，去掉侧边模式专用的文案分配偏差。
- `CenterCardLayoutPolicy`
  统一横向 / 竖向中心卡片长度映射与歌词布局参数。
- `CapsuleLayoutManager`
  统一四种吸附模式的主轴、可用空间、详细页展开方向。
- `MainWindow.xaml`
  把中心卡片视图组织成可按方向重排的统一结构。
- `MainWindow.xaml.cs`
  只做状态绑定、方向切换、动画方向切换、拖拽长度交互与悬浮层路由。

## 8. 测试与验证

### 8.1 自动化测试

至少补齐以下验证：

- 顶部模式与底部模式共享同一套中心卡片状态切换规则
- 左右吸附的中心卡片长度映射来自同一比例参数
- 左右吸附歌词动画方向为自下而上
- 左右吸附详细态包含和底部一致的控件能力
- 左右吸附的详细页 / 悬浮页朝外侧展开
- 胶囊拖拽位置被限制在可恢复范围内

### 8.2 视觉验证

除了测试，还必须做实际运行后的人工截图检查。

左右吸附至少要检查：

- 图标是否保持正向
- 常态歌词是否真实出现在中心卡片内
- 歌词是否从下往上进入
- 详细态是否显示歌曲名、歌手、播放器控件
- 详细页是否出现在胶囊外侧
- 长度调节后中心卡片主体是否同步变长

## 9. 成功标准

本次工作完成后，用户应当能把四种吸附模式理解成同一个产品，而不是四套不同完成度的 UI。

成功标准：

1. 顶部吸附在功能和交互上与底部胶囊一致，只保留顶部默认参数差异。
2. 左右吸附在功能和参数上与底部胶囊一致，只把内容改为竖排。
3. 左右吸附中心卡片常态真实显示歌词，且歌词从下往上滑动。
4. 左右吸附悬浮详细态显示完整播放器与歌曲信息，不再缺项。
5. 上 / 左 / 右三种吸附的默认长度继承顶部参数，但用户后续仍可手动调节长度。
6. 详细页和悬浮页始终朝胶囊外侧展开。
7. 实现不再依赖“侧边模式单独打一套补丁”的结构。

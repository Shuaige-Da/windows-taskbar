# 胶囊视觉融合版设计说明

> 状态：已作为 DynamicIslandBar V1 视觉基线落地。对应 GitHub 分支为 `win-ui1.0` 和 `win-ui1.0-优化版`，后续视觉工作应在此基础上迭代。

## 目标

把当前胶囊升级为「A+B+C 融合版」：默认观感达到可发行软件的稳定质感，同时保留灵动岛式的交互反馈和可记忆的流光特效。

## 设计方向

### 默认视觉：Windows 玻璃底

胶囊主体采用深色 Mica/Acrylic 质感，而不是纯黑色块。背景应包含半透明深色、轻微内发光、柔和边框和投影，让它贴在桌面或任务栏上时有真实层次。默认状态不能太花，优先保证长期使用不累眼。

### 交互视觉：灵动岛扩展

应用图标悬浮时向右扩展成小胶囊，显示应用名称和状态。扩展动效应短、顺滑、克制，避免影响系统区和时间区域。中心区域继续承接当前窗口信息，让胶囊中部不空。

### 品牌记忆点：图标同色流光

流光使用当前悬浮或激活应用图标的主色调。线条比现在更粗、更清晰，但默认透明度保持克制；悬浮时增强，离开后淡回。这样既有个性，也不会显得像游戏外挂。

## 布局规则

底部模式应覆盖系统任务栏区域，视觉上成为新的主任务栏。顶部模式应紧贴屏幕顶部，胶囊宽度收窄，但保留应用区、当前窗口区、WiFi、音量、电池、时间和方格按钮。

应用图标区域、中心摘要区域、系统区域保持三段式结构。主栏只展示有限数量应用，超出应用进入收纳图标和方格面板，避免挤压时间和系统信息。

## 可升级性

本次实现只改默认主题和现有交互，不引入复杂主题编辑器。代码上应为后续「用户自定义胶囊风格、背景图片、流光强度、边框样式」留出入口，优先通过主题管理类和少量可复用样式完成。

## 验收标准

1. 胶囊默认外观明显比当前更精致，不再像单纯黑色长条。
2. 应用悬浮扩展、中心摘要、系统区在底部和顶部模式下都不互相挤压。
3. 流光颜色跟随当前应用主色，线条更粗、更可见。
4. 底部模式继续覆盖系统任务栏，顶部模式继续紧贴屏幕顶部。
5. 项目可以通过现有测试和 `dotnet build`。

## V1 实现记录

V1 已实现该融合方向的核心目标：

- 默认视觉采用透明玻璃态胶囊，保留深色主体、柔和边缘和流光边框。
- 浮出面板统一接入胶囊主题，包括 WiFi、音量、运行列表、更多应用和权限提示。
- 应用悬浮说明层改为覆盖层，不再挤压后续图标。
- 流光采用跑马灯式边框，并支持亮度、粗细、速度调节。
- 悬浮或激活应用时，流光颜色会跟随应用图标主色。
- 胶囊支持底部任务栏模式和顶部灵动岛模式。
- 外观控制已包含透明度、阴影、胶囊粗细、胶囊长度等基础自定义项。

V1 之后的视觉方向：

- 把右键菜单中的外观项迁移到更完整的可视化设置页。
- 增加背景图片上传和预览。
- 增加更多风格预设，但避免在 V1.1 包装阶段引入过重主题系统。
- 保持“玻璃底 + 灵动岛交互 + 同色流光”作为默认产品识别。
